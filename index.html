01.About Me

I'm an Infrastructure/DevOps Engineer who treats infrastructure as a codebase. I keep Terraform modules small, well-named, and commented, so on-call engineers can reason about changes under pressure.

I usually end up owning public cloud foundations, VPC layouts, EKS/GKE clusters, and CI/CD deployment paths. I have a habit of building small Python tools around AWS and GCP APIs to automate routine tasks and eliminate toil.

Security is folded into my daily work—wiring IAM roles and policy checks into CI so least privilege is a default, not an afterthought.
